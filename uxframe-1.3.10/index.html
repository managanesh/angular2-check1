Getting Started

John Deere User Experience Standards are available as a single download named UXFrame. This framework contains all of the necessary code to begin writing a web application that follows internal standards. New applications can be easily created by following provided examples and documentation, and existing applications can be updated progressively.

UXFrame is our canonical implementation of the John Deere visual standards. Although, it is certainly possible to achieve the same aesthetic using different tools or code, and using our implementation is in no way required. If using a custom implementation fits team need better, that approach should be preferred, and our implementation should serve as a style guide.

However, by using our framework, developers can spend less time recreating UI elements and layouts and more time working on an application. Additionally, components built using the framework can be used across applications and reintegrated into the standards for other teams to use.

UXFrame is currently under active development by a small team of developers on the IT UX team. We are always using, debugging, and improving our code. As with any project, it is always important to include proper front-end testing when using our code. We hope you will engage with our team to improve this framework.

We've also included several open source libraries that can be used along side our code. Basic documentation and examples for these libraries are included.

For questions, comments, thoughts, ideas, and contributions please contact Ben Seefeldt or Craig Newkirk.

Downloads

Standard Distribution

The standard download for UXFrame includes everything a standard developer will need to get started using the framework. It includes a single CSS and JavaScript file to import into a project, as well as the John Deere Sans font file and necessary images. In addition to these necessary resources it also includes open source libraries, developer documentation and a handful of coded examples. If you're new to this tool, start here.

Minimal Distribution

The miminal download removes the documentation and examples and includes only the content necessary to run a project. This should be used by developers who are comfortable using UXFrame, or projects that are already using the framework.

Advanced Distribution

UXFrame is built using less and grunt. The source files to rebuild the standards are included in the advanced download. If your project uses these technologies, or if you are interested in extending the current standards, this distribution may be helpful. This also includes examples of features that are currently in developer preview.
