Complete Page

If you're starting a new project we start with an empty HTML document. We recommend the use of HTML5 and CSS3 for new projects. Additionally, using the .jd class on the <html> element applies our styles to the entire document. This resets default stying and applies John Deere Standards to all default HTML elements and may cause problems if applied to existing pages without some refactoring. 

<!DOCTYPE html>
<html lang="en" class="jd">
<head>
    <script src="js/uxframe-version.min.js"></script>  
    <link rel="stylesheet" href="css/uxframe-version.min.css">
    <link rel="shortcut icon" href="img/favicon.ico" type="image/x-icon" />
</head>
    ...
</html>

Individual Elements

While it is not recommended, it is possible to use individual components of the John Deere Standards without "taking the dive" and using the entire setup. The components documented below can be wrapped in a div with the .jd class applied. 

<div class="jd">
    ... components ...
</div>

Layout

One of the most important pieces of making a fully functional website is page layout. Standard HTML provides powerful tools for laying out and reflowing text. As part of the standads, we provide a set of tools for using percentage width div elements to layout a page. We strongly recommend this approach over other alternatives such as using table elements. Relying on div structure is in keeping with a modern approach to website design, and standard across many web frameworks. While other approaches could feasibly be used, the standards have not been designed with these in mind, and it is likely there will be unforseen complications.

Layout is done with nested divs forming row and column structures. Each column must be inside a containing row. Content always lives within a column. Multiple levels of rows and columns can be nested to create more complex structures. Columns are given a width using fractional notation. For example .col-1-3 creates a column with 33% width. .col-1 is used for 100% width. Fractions ranging up to sixths are available, always in the most simplified form. (i.e. 2-3 not 4-6.) The example below uses colored divs to illustrate this. (Note that inline height or padding is not necessary and is used only for demonstration, content and panels will correctly set these styles.) 



                    

                        

                            
 
                            
 
                            
 
                        

                        

                            

                            

                            

                        

                        

                            

                                

                                    

                                    

                                

                            

                        

                    

                    
Advanced Layout

                    
 
                        

                        Shrewd observers will notice additional classes being used to control
                        margins and padding in certain elements. In the panel section, the class
                        .clear-panel was mentioned for controlling the bottom
                        margin of the panel element. Another example is the .last
                        class on the final paragraph element.
                        However, most elements including lists and navigational tools
                        have margins or padding on the bottom and/or right side of the element.
                        In order to maintain compliance with IE7, we have avoided the use of the
                        last-child psuedo-selector in our CSS. This means some
                        spacing elements should be manually cleared. Below is a list of classes
                        that can be used to modify these elements.
                        

                        

                            
.clear-right

                            
Clears right padding and right margin on an element.

                            
.clear-right-margin

                            
Clears only right margin on an element.

                            
.clear-bottom

                            
Clears bottom padding and bottom margin on an element.

                            
.clear-bottom-margin

                            
Clears only bottom margin on an element.

                            
.clear-panel

                            
Equivalent to clear-bottom-margin.

                        

                        

                        An additional tool for placing content is inline layout. The provided
                        inline classes are frequently used to place input elements or buttons
                        in a single row. Currently, width of some of these elements
                        is controlled by inline width styles, but this may be changed
                        in the future. A few basic examples can be seen below.

                    content. They provide clear distinction between different pieces

                    Panels are used on this page to hold different sections of documentation.
                    They are placed within a column element.
                    

                    

                    The margin at the bottom of the panel can be cleared by using the
                    .clear-panel class.

                    A variety of navigation elements are provided by default. These cover most of the 
                    common types of page navigation used in modern web pages. In addition to page
                    navigation, the top nav element can also hold a page search bar, notification icon,
                    or shopping cart status.

                        be a good way to provide your users with a persistent way of

Left Navigation

                    

                        

                        A left navigation element can be used to provide nested navigation
                        for a structured site with 1-2 levels. Our nav is based on a
                        nested list structure. Elements with no children are a single
                        <li> element, while nested lists are a second
                        <ul> within a parent <li>.
                        This structure can be seen in the example below.
                        

                        

                        The selection structure is controlled by the classes .selected
                        and .selected-parent. These class labels are applied
                        to add a green selected bar and expand child elements respectively.
                        

                        

                        Note that in the example below the .interactive class is 
                        applied to the element. This allows the user to traverse between
                        top level items without navigating to a new page. With this class removed
                        active states should be managed on a 'per-page' basis. See examples 
                        for non-interactive navs.
                        

                        

                        When laying out a page, the left navigation element can be placed
                        inside a .col-1-6 div and the rest of the page
                        content is inside a .col-5-6 div. This structure
                        is used on this page and many of the demo pages.

                    Standards provides stylings for all standard HTML header elements
                    <h1> to <h6>. Additionally,
                    for inline code styled in the same fashion as a header classes
                    .h1 to .h6 are also provided. Styling
                    is also given on the <p> tag, which should be
                    used for all body content. The final <p> in a
                    sequence should include the .last class.
                    

                    

                    In addition to standard tags such as
                    <b>,
                    <i>, and
                    <s> we provide the
                    .secondary-info class
                    used for formatting secondary info.
                    Links contained in <a> tags are colored
                    John Deere green.

Lists

                    

                    Lists can be presented in several ways depending on the use case.
                    These styles are controlled by classes on the <ul>
                    element. Note that lists by default have bottom margins so they
                    can be used inline with paragraphs. If a list is the final element
                    in a panel div it should have a .clear-bottom class
                    added to clear this margin.
                    

                    

                    Lists can also have bullets, or numbers. (Note that the standard
                    list is left aligned with the edge of a paragraph, but the
                    .indented class provides a slight indent when
                    stylistically necessary.) Additionally, we provide
                    several types of description list including .standard,
                    .horizontal and .horizontal.wide.

Buttons

                    

                    Two main types of buttons are available, call to action and secondary.
                    Call to action buttons are used for the primary task of a page, or
                    the preferred 'next step' for a user. Secondary buttons are used
                    for additional tasks or actions that a user may take.
                    Buttons may also be 'disabled' to show a user actions that may be
                    dependent on other tasks, such as completing a form.
                    The buttons additionally come in a smaller style which can be
                    used for tables or inline on forms.
                    

                    

                    Both button types can also be used as a dropdown, which can be
                    useful for including several actions without cluttering a page
                    with too many buttons. Many times these can be used to provide
                    an additional 'actions' menu for miscellaneous tasks.
                    

                    

                    There is a final button style for invisible buttons .invis
                    which renders similarly to a link. This can be used to create a dropdown
                    menu inline or without creating a button around it. Other than this specific
                    use, the invis button should usually not be used.

Tables

                    

                    Tables should not be used for full page layout, but rather
                    should hold table formatted data similar to a spreadsheet.
                    We provide a common style for tables along with common elements
                    like checkboxes that are frequently used to gather user input.
                    

                    

                    For more advanced functionality, we support the use of the 
                    DataTables plugin. 
                    This library depends upon JQuery, which is also included in our
                    set of supported libraries. We provide CSS for the basic usage
                    of this plugin to match standards.

Header

                    

                    Most webpages will have a static header at the top of the page,
                    many times including a title and maybe a user dropdown. The header
                    will include standardized John Deere branding. The header can be
                    customized to fit different information, and different configurations
                    can be found in the demo files. As a general rule, every page should
                    include a header.

Floating Alerts

                    

                    In some cases, a floating alert may be used rather than a permanent static alert.
                    These are frequently used to indicate save or loading status. These alerts
                    overlay content and are fixed in position on the page. Note that alerts 
                    do not automatically manage screen position or layering. This means that it is
                    possible to 'stack' alerts on each other intadvertently. Developers should ensure
                    that an existing float alert is properly removed before adding another.

<!DOCTYPE html>

Site Style

                    

                    The standard style used is also referred to as 'App Style'. This
                    general style has a gray background, dark footer, white navbar,
                    and white panels. This is the style typically used in a webpage
                    that serves as an application.
                    

                    

                    An alternative set of stylings may be used when the webpage is used
                    in some cases. This set is referred to as 'Site Style' and are implemented
                    in a similar fashion to the C&F stylings. The .site-style
                    class can be added to any element, but typically the <html>
                    element.
                    


<!DOCTYPE html>
<html lang="en" class="jd site-style">
    ...
</html>


                    

                    Site styling changes the background color, footer color, panel
                    shadow, and top navigation color.
                    

                


                

                    
JavaScript

                    

                    We provide a small amount of JavaScript which handles some layout
                    pieces and minimal interactions. In the advanced distribution
                    individual source files are separated by component. Additional custom
                    components can be added following provided template. Note that
                    unlike the library section, this JavaScript has no external dependencies.
                    

                    

                        

                            

                            Our primary base object is $UXF or UXFrameHelper
                            both of which refer to the same object. All accessible functions
                            are accessed through this object.
                            

                            

                            By default, we run configuration and bind events to all of our 
                            elements present on the page. To disable this behavior we 
                            specify after the inclusion of the UXFrame script. Such as:
                            


<script src="../js/uxframe-1.3.4.min.js"></script>
<script> $UXF.defaultBinds = false; </script>


                            

                            Binding objects can be done manually through a series of functions.
                            Registration can be done for either all components, or individual
                            components
                            


$UXF.registerAll();
$UXF.Register.Accordion();
var a = document.getElementById('#new');
$UXF.Register.Accordion(a);


                            

                            For specific registration functions (e.g.: $UXF.Register.*)
                            an optional argument registers only the passed element, or its children.
                            Note that this will not rebind elements that have already been associated
                            with a component.
                            

                        

                        

                            

                            Currently accordions support deregistering functionality. This is
                            done in same same way that registering works. For example, given
                            an element a we can run the function
                            $UXF.Deregister.Accordion(a); to remove the previously
                            set event watchers. Future releases will support additional deregister functions.
                            
                            

                            

                            Some components allow for direct JavaScript manipulation rather than
                            relying on events. For example, modals must be triggered directly.
                            (Forgoing detailed documentation, examining source should provide a
                            good idea of which functions are available.) To construct a modal object,
                            we can provide the function either a DOM object, or an element ID.
                            The .toggle() function changes state.
                            


var m1 = $UXF.Modal('my-modal');
var m2 = $UXF.Modal(dom_element);
m2.toggle();


                            

                            Additionally, we provide a series of helper functions to ensure
                            compatibility with IE8+. These are widely used throughout our 
                            components.
